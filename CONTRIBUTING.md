# Contributing to SaleScout

Thank you for your interest in contributing to SaleScout!

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/SaleScout.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Run tests (if available)
6. Commit: `git commit -m "Add: your feature description"`
7. Push: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Code Guidelines

- **Backend (Python)**: Follow PEP 8, add docstrings, use type hints where appropriate
- **Frontend (React)**: Use functional components, hooks, and clean JSX
- **Commits**: Use clear, descriptive commit messages

## Pull Request Process

1. Update README.md if needed
2. Ensure all services run without errors
3. Test your changes with Docker Compose
4. Describe your changes clearly in the PR

## Reporting Issues

- Use GitHub Issues
- Include steps to reproduce
- Add screenshots if relevant
- Specify your environment (OS, Docker version, etc.)

## Questions?

Open a GitHub Discussion or create an issue.
